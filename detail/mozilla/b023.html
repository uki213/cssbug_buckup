固定配置されたリストアイテム要素内で文字間隔を変えると要素全体が動く(Moz1.0RC3)

症状

幅を相対単位の値で指定し、かつ固定配置にしたリストアイテム要素（li要素など、 display: list-item; が指定された要素）内で、hover擬似クラスを用いてletter-spacingプロパティの値を変えると、マウス等でポイントした瞬間に要素全体が移動する。

例示

<style type="text/css">
li.a {
  position: fixed;
  width: 10em;
  left: 30px;
  top: 10px;
  border: 2px solid blue;
}
a.b:hover {
  letter-spacing: 1em;
}
</style>

ページ左上に固定配置されたli要素内にある2つのリンク上でマウスポインタを何度も行き来させてみてください。

修正状況

Moz1.0では標準・互換モードともに不具合の発生が確認されました。N6.2.3では標準・互換モードともに不具合は発生しませんでした。
