@mediaブロックの括弧の対応関係を正しく認識しない(4.0)

症状

WinIE4は@mediaキーワードを無視するが、そのブロックの中括弧の対応関係を正しく認識せず最初に現れる閉じ中括弧までを@mediaのブロックと認識する。

例示

<style type="text/css">
@media screen {
  div#t1 { color: blue; }
  div#t2 { color: blue; }
}
</style>

<div id="t1">青色</div>
<div id="t2">青色</div>
青色1
青色2

@mediaのブロックを無視する場合、両方のdiv要素の前景色は青色にならないはずです。WinIE4.0は、div#t1のブロックの終了を示す括弧で@mediaのブロックも終了すると判断するため、id属性が「t2」のdiv要素の前景色は青色になってしまうそうです。

修正状況

WinIE4.0でこの不具合が発生するそうです。
